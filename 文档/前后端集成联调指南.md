### 1.1 后端API地址配置

**开发环境后端地址**: `http://localhost:8080`

### 2.2 前端API代理配置

修改 `vite.config.js`，添加代理配置解决开发环境跨域问题：

```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { fileURLToPath, URL } from 'node:url'

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  },
  server: {
    port: 3000,
    open: true,
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '/api')
      }
    }
  }
})
```

### 2.3 前端请求封装

`src/utils/request.js` 已封装完整的请求拦截器：

```javascript
import axios from 'axios'
import { useUserStore } from '@/stores/user'

const service = axios.create({
  baseURL: '/api',
  timeout: 10000
})

// 请求拦截器：自动添加Token
service.interceptors.request.use(
  config => {
    const userStore = useUserStore()
    if (userStore.token) {
      config.headers['Authorization'] = `Bearer ${userStore.token}`
    }
    return config
  },
  error => Promise.reject(error)
)

// 响应拦截器：统一处理401、403
service.interceptors.response.use(
  response => {
    const res = response.data
    if (res.code === 200) {
      return res.data
    }
    return Promise.reject(new Error(res.message))
  },
  error => {
    if (error.response?.status === 401) {
      const userStore = useUserStore()
      userStore.logout()
      window.location.href = '/login'
    }
    return Promise.reject(error)
  }
)

export default service
```

### 2.4 后端跨域配置

后端 `WebMvcConfig.java` 已配置CORS：

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")
                .allowedOriginPatterns("*")
                .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
                .allowedHeaders("*")
                .allowCredentials(true)
                .maxAge(3600);
    }
}
```

## 五、常见报错解决方案

### 5.1 跨域问题 (CORS Error)

**错误信息**：
```
Access to XMLHttpRequest at 'http://localhost:8080/api/...' from origin
'http://localhost:3000' has been blocked by CORS policy
```

**解决方案**：
1. 后端已配置CORS，确保`WebMvcConfig.java`正确
2. 开发环境使用代理，修改`vite.config.js`
3. 检查后端是否设置了正确的CORS响应头

### 5.2 401未授权错误

**错误信息**：
```json
{"code": 401, "message": "请先登录"}
```

**解决方案**：
1. 检查请求是否携带了Token：`Authorization: Bearer {token}`
2. 检查Token是否过期（JWT默认24小时）
3. 检查`AuthInterceptor`是否正确解析Token
4. 前端登录后必须保存Token到store

### 5.3 403权限不足错误

**错误信息**：
```json
{"code": 403, "message": "权限不足"}
```

**解决方案**：
1. 检查用户角色是否匹配接口要求
2. 确保`@RequiresRoles`注解配置正确
3. 管理员接口需要admin角色

### 5.4 参数校验错误

**错误信息**：
```json
{"code": 400, "message": "奖励金额必须大于0"}
```

**解决方案**：
1. 检查请求参数是否完整
2. 检查参数格式是否正确（日期、金额等）
3. 查看`GlobalExceptionHandler`捕获的校验异常

### 5.5 事务失效问题

**问题描述**：资金操作部分成功部分失败，数据不一致

**解决方案**：
1. 确保方法有`@Transactional`注解
2. 确保数据库表使用InnoDB引擎（支持事务）
3. 检查是否手动catch了异常导致事务未回滚

### 5.6 状态不流转问题

**问题描述**：任务状态无法从pending变为ongoing

**解决方案**：
1. 检查任务当前状态是否正确
2. 检查`TaskServiceImpl`中的状态校验逻辑
3. 确保并发操作使用乐观锁（version字段）

### 6.2 前端路由权限控制

```javascript
// src/router/index.js
const routes = [
  { path: '/login', name: 'Login', component: Login },
  { path: '/register', name: 'Register', component: Register },
  { path: '/', component: AppLayout,
    children: [
      { path: '', name: 'Home', component: Home },
      { path: 'publish', name: 'Publish', component: Publish,
        meta: { roles: ['requester', 'admin'] } },
      { path: 'my-tasks', name: 'MyTasks', component: MyTasks },
      { path: 'admin', name: 'Admin', component: Admin,
        meta: { roles: ['admin'] } },
    ]
  }
]

// 路由守卫进行权限校验
router.beforeEach((to, from, next) => {
  const userStore = useUserStore()
  if (to.meta.roles && !to.meta.roles.includes(userStore.role)) {
    next('/')
  } else {
    next()
  }
})
```
