# 数据库设计文档

## 1. 数据库概述

### 1.1 数据库信息

| 项目 | 内容 |
|------|------|
| 数据库名 | campus_master |
| 数据库类型 | MySQL 8.0 |
| 字符集 | utf8mb4 |
| 排序规则 | utf8mb4_unicode_ci |

### 1.2 设计原则

- 遵循第三范式（3NF），减少数据冗余
- 主键统一使用BIGINT自增
- 统一使用逻辑删除（deleted字段）
- 统一使用时间戳（create_time, update_time）
- 金额统一使用DECIMAL(10,2)

## 3. 表结构设计

### 3.1 用户表 (user)

```sql
CREATE TABLE `user` (
    `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '用户ID',
    `username` VARCHAR(50) NOT NULL COMMENT '用户名',
    `phone` VARCHAR(20) NOT NULL COMMENT '手机号',
    `password` VARCHAR(255) NOT NULL COMMENT '密码',
    `role` VARCHAR(20) NOT NULL COMMENT '角色：requester/helper/admin',
    `credit_score` INT DEFAULT 100 COMMENT '信用分 0-100',
    `status` TINYINT DEFAULT 1 COMMENT '状态：1正常，0冻结',
    `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除：0未删除，1已删除',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_phone` (`phone`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

**说明**：
- phone字段唯一索引，用于登录验证
- credit_score默认100分，上限100，下限0
- role字段限制用户类型

### 3.2 任务表 (task)

```sql
CREATE TABLE `task` (
    `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '任务ID',
    `title` VARCHAR(100) NOT NULL COMMENT '任务标题',
    `description` TEXT COMMENT '详细描述',
    `category` VARCHAR(50) COMMENT '任务分类',
    `reward` DECIMAL(10,2) NOT NULL COMMENT '奖励金额',
    `service_fee` DECIMAL(10,2) NOT NULL COMMENT '服务费（奖励*5%）',
    `status` VARCHAR(20) NOT NULL DEFAULT 'pending' COMMENT '状态：pending/ongoing/pending_review/completed/cancelled/disputed',
    `deadline` DATETIME COMMENT '截止时间',
    `location` VARCHAR(200) COMMENT '任务地点',
    `contact_info` VARCHAR(100) COMMENT '联系方式',
    `view_count` INT DEFAULT 0 COMMENT '浏览次数',
    `proof_images` TEXT COMMENT '完成凭证图片',
    `requester_id` BIGINT NOT NULL COMMENT '需求方用户ID',
    `helper_id` BIGINT COMMENT '接单方用户ID',
    `accept_time` DATETIME COMMENT '接单时间',
    `submit_time` DATETIME COMMENT '提交时间',
    `complete_time` DATETIME COMMENT '完成时间',
    `version` INT DEFAULT 0 COMMENT '乐观锁版本号',
    `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`),
    INDEX `idx_requester` (`requester_id`),
    INDEX `idx_helper` (`helper_id`),
    INDEX `idx_status` (`status`),
    INDEX `idx_category` (`category`),
    CONSTRAINT `fk_task_requester` FOREIGN KEY (`requester_id`) REFERENCES `user` (`id`),
    CONSTRAINT `fk_task_helper` FOREIGN KEY (`helper_id`) REFERENCES `user` (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='任务表';
```

**说明**：
- status字段使用VARCHAR便于扩展状态
- version字段用于乐观锁，防止并发更新
- service_fee = reward * 0.05

### 3.3 账户表 (account)

```sql
CREATE TABLE `account` (
    `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '账户ID',
    `user_id` BIGINT NOT NULL COMMENT '用户ID',
    `balance` DECIMAL(10,2) DEFAULT 0.00 COMMENT '可用余额',
    `frozen_amount` DECIMAL(10,2) DEFAULT 0.00 COMMENT '冻结金额',
    `total_income` DECIMAL(10,2) DEFAULT 0.00 COMMENT '累计收入',
    `total_expense` DECIMAL(10,2) DEFAULT 0.00 COMMENT '累计支出',
    `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_user_id` (`user_id`),
    CONSTRAINT `fk_account_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='账户表';
```

**说明**：
- 一个用户只有一个账户
- frozen_amount冻结金额用于任务担保
- total_income/total_expense用于统计

### 3.4 交易记录表 (transaction)

```sql
CREATE TABLE `transaction` (
    `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '交易ID',
    `user_id` BIGINT NOT NULL COMMENT '用户ID',
    `task_id` BIGINT COMMENT '关联任务ID',
    `type` VARCHAR(20) NOT NULL COMMENT '交易类型：recharge/freeze/unfreeze/payment/income/refund',
    `amount` DECIMAL(10,2) NOT NULL COMMENT '交易金额',
    `balance_before` DECIMAL(10,2) COMMENT '变动前余额',
    `balance_after` DECIMAL(10,2) COMMENT '变动后余额',
    `remark` VARCHAR(200) COMMENT '备注',
    `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    PRIMARY KEY (`id`),
    INDEX `idx_user_time` (`user_id`, `create_time`),
    INDEX `idx_task` (`task_id`),
    CONSTRAINT `fk_transaction_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`),
    CONSTRAINT `fk_transaction_task` FOREIGN KEY (`task_id`) REFERENCES `task` (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='交易记录表';
```

**说明**：
- 记录所有资金变动，保证可追溯性
- balance_before/after记录变动前后余额

### 3.5 评价表 (review)

```sql
CREATE TABLE `review` (
    `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '评价ID',
    `task_id` BIGINT NOT NULL COMMENT '任务ID',
    `reviewer_id` BIGINT NOT NULL COMMENT '评价人ID',
    `reviewee_id` BIGINT NOT NULL COMMENT '被评价人ID',
    `rating` TINYINT NOT NULL COMMENT '评分1-5星',
    `content` VARCHAR(500) COMMENT '评价内容',
    `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    PRIMARY KEY (`id`),
    INDEX `idx_task` (`task_id`),
    INDEX `idx_reviewer` (`reviewer_id`),
    INDEX `idx_reviewee` (`reviewee_id`),
    CONSTRAINT `fk_review_task` FOREIGN KEY (`task_id`) REFERENCES `task` (`id`),
    CONSTRAINT `fk_review_reviewer` FOREIGN KEY (`reviewer_id`) REFERENCES `user` (`id`),
    CONSTRAINT `fk_review_reviewee` FOREIGN KEY (`reviewee_id`) REFERENCES `user` (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='评价表';
```

**说明**：
- 双向评价，任务完成后双方互评
- 一个任务最多产生2条评价

### 3.6 通知表 (notification)

```sql
CREATE TABLE `notification` (
    `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '通知ID',
    `user_id` BIGINT NOT NULL COMMENT '用户ID',
    `task_id` BIGINT COMMENT '关联任务ID',
    `type` VARCHAR(20) NOT NULL COMMENT '通知类型',
    `title` VARCHAR(100) NOT NULL COMMENT '通知标题',
    `content` TEXT COMMENT '通知内容',
    `is_read` TINYINT DEFAULT 0 COMMENT '是否已读：0未读，1已读',
    `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    PRIMARY KEY (`id`),
    INDEX `idx_user_read` (`user_id`, `is_read`),
    INDEX `idx_create_time` (`create_time`),
    CONSTRAINT `fk_notification_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`),
    CONSTRAINT `fk_notification_task` FOREIGN KEY (`task_id`) REFERENCES `task` (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='通知表';
```

**说明**：
- 记录用户所有通知
- is_read字段支持未读数查询

### 3.7 争议表 (dispute)

```sql
CREATE TABLE `dispute` (
    `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '争议ID',
    `task_id` BIGINT NOT NULL COMMENT '关联任务ID',
    `complainant_id` BIGINT NOT NULL COMMENT '申诉人ID',
    `reason` VARCHAR(500) NOT NULL COMMENT '申诉原因',
    `status` VARCHAR(20) DEFAULT 'pending' COMMENT '状态：pending/resolved',
    `result` VARCHAR(50) COMMENT '处理结果',
    `remark` VARCHAR(500) COMMENT '处理备注',
    `handler_id` BIGINT COMMENT '处理人ID（管理员）',
    `resolve_time` DATETIME COMMENT '处理时间',
    `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    PRIMARY KEY (`id`),
    INDEX `idx_task` (`task_id`),
    INDEX `idx_complainant` (`complainant_id`),
    INDEX `idx_status` (`status`),
    CONSTRAINT `fk_dispute_task` FOREIGN KEY (`task_id`) REFERENCES `task` (`id`),
    CONSTRAINT `fk_dispute_complainant` FOREIGN KEY (`complainant_id`) REFERENCES `user` (`id`),
    CONSTRAINT `fk_dispute_handler` FOREIGN KEY (`handler_id`) REFERENCES `user` (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='争议表';
```

**说明**：
- 只有任务相关方可以发起争议
- 管理员处理争议

### 3.8 任务日志表 (task_log)

```sql
CREATE TABLE `task_log` (
    `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '日志ID',
    `task_id` BIGINT NOT NULL COMMENT '任务ID',
    `operator_id` BIGINT COMMENT '操作人ID',
    `action` VARCHAR(50) NOT NULL COMMENT '操作类型',
    `description` VARCHAR(200) COMMENT '操作描述',
    `old_status` VARCHAR(20) COMMENT '原状态',
    `new_status` VARCHAR(20) COMMENT '新状态',
    `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    PRIMARY KEY (`id`),
    INDEX `idx_task` (`task_id`),
    INDEX `idx_create_time` (`create_time`),
    CONSTRAINT `fk_task_log_task` FOREIGN KEY (`task_id`) REFERENCES `task` (`id`),
    CONSTRAINT `fk_task_log_operator` FOREIGN KEY (`operator_id`) REFERENCES `user` (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='任务日志表';
```

**说明**：
- 记录任务的完整生命周期
- 用于问题追溯和审计

## 5. 索引设计说明

### 5.1 索引使用场景

| 查询场景 | 使用索引 | 说明 |
|----------|----------|------|
| 按手机号登录 | uk_phone | 唯一索引 |
| 查看我的发布任务 | idx_requester | 需求方查询 |
| 查看我的接单任务 | idx_helper | 接单方查询 |
| 任务大厅筛选 | idx_status, idx_category | 状态+分类组合 |
| 未读通知查询 | idx_user_read | 用户+未读状态 |
| 交易记录查询 | idx_user_time | 用户+时间排序 |

### 5.2 索引优化建议

```sql
-- 复合索引优化
CREATE INDEX idx_status_category ON task(status, category);

-- 覆盖索引优化查询
CREATE INDEX idx_user_time_cover ON transaction(user_id, create_time, type, amount);
```

## 6. 数据字典

### 6.1 用户角色 (user.role)

| 值 | 说明 |
|----|------|
| requester | 需求方，可发布任务 |
| helper | 接单方，可承接任务 |
| admin | 管理员，全权限 |

### 6.2 任务状态 (task.status)

| 值 | 说明 | 可执行操作 |
|----|------|------------|
| pending | 待接单 | 接单、取消 |
| ongoing | 进行中 | 提交凭证 |
| pending_review | 待验收 | 验收、发起争议 |
| completed | 已完成 | 评价 |
| cancelled | 已取消 | - |
| disputed | 争议中 | 管理员处理 |

### 6.3 交易类型 (transaction.type)

| 值 | 说明 |
|----|------|
| recharge | 充值 |
| freeze | 冻结 |
| unfreeze | 解冻 |
| payment | 支付 |
| income | 收入 |
| refund | 退款 |

### 6.4 通知类型 (notification.type)

| 值 | 说明 |
|----|------|
| task_accept | 任务被接单 |
| task_submit | 任务已提交 |
| task_complete | 任务已完成 |
| task_cancel | 任务已取消 |
| dispute_submit | 争议已提交 |
| dispute_resolve | 争议已处理 |
| review_received | 收到新评价 |
| system | 系统通知 |
