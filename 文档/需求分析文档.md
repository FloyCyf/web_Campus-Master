# 需求分析文档

## 1. 项目概述

### 1.1 项目名称

校园"万事达"互助众包任务平台（Campus Master）

### 1.2 项目背景

随着高校校园信息化建设的不断推进，学生日常生活中存在大量碎片化的互助需求，如代取快递、代买东西、文件打印等。传统的方式依靠朋友圈、QQ群等非平台化渠道，信息分散、信任机制缺失、交易无保障。

本平台旨在构建一个安全、可信的校园互助众包系统，通过平台担保交易、信用评价、任务追踪等功能，解决校园互助场景中的信任和安全问题。

### 1.3 项目目标

1. 为校园用户提供一个便捷、高效的互助任务发布与接单平台
2. 通过资金担保和信用评分机制，建立校园互助的信任体系
3. 提供实时通知和任务状态追踪，提升用户体验
4. 引入管理员角色，保障平台健康运行

## 2. 用户分析

### 2.1 用户角色

| 角色 | 说明 | 主要功能 |
|------|------|----------|
| 需求方(requester) | 有互助需求的用户 | 发布任务、充值账户、验收任务、发起争议 |
| 接单方(helper) | 提供互助服务的用户 | 承接任务、提交凭证、收取报酬 |
| 管理员(admin) | 平台运营人员 | 审核任务、处理争议、统计报表、用户管理 |

### 2.2 用户特征

- **年龄群体**：18-25岁大学生
- **使用习惯**：移动端为主，PC端为辅
- **核心诉求**：便捷、安全、省时
- **信任顾虑**：对方是否可信、资金是否安全、任务能否完成

### 3.2 详细功能需求

#### 3.2.1 用户模块

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 用户注册 | P0 | 手机号+密码+角色注册，支持需求方/接单方 |
| 用户登录 | P0 | 手机号+密码登录，返回JWT令牌 |
| 个人信息管理 | P1 | 查看和修改个人信息 |
| 账户余额查询 | P1 | 查看账户余额、冻结金额、可用金额 |
| 信用评分查看 | P1 | 查看个人信用评分和评价记录 |

#### 3.2.2 任务模块

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 发布任务 | P0 | 需求方发布互助任务，设定奖励金额和截止时间 |
| 浏览任务 | P0 | 查看所有待接单任务，支持分类筛选 |
| 接单任务 | P0 | 接单方接受任务，进入进行中状态 |
| 任务追踪 | P1 | 查看任务当前状态和时间线 |
| 提交凭证 | P0 | 接单方上传任务完成证明 |
| 确认验收 | P0 | 需求方确认任务完成，触发资金结算 |
| 取消任务 | P1 | 需求方在接单前取消，全额退款 |
| 发起争议 | P1 | 任务完成后有异议可发起争议 |

#### 3.2.3 账户模块

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 账户充值 | P1 | 用户充值到平台账户 |
| 资金冻结 | P0 | 发布任务时冻结奖励金额 |
| 资金解冻 | P0 | 取消任务时解冻资金 |
| 资金结算 | P0 | 任务完成后从冻结转入接单方账户 |
| 交易记录 | P1 | 查看账户所有收支记录 |

#### 3.2.4 评价模块

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 发布评价 | P1 | 任务完成后双方互评，1-5星评分 |
| 查看评价 | P1 | 查看他人对自己或他人的评价 |
| 信用评分更新 | P1 | 根据评价计算用户信用分 |

#### 3.2.5 通知模块

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 实时通知推送 | P2 | WebSocket实时推送任务状态变更 |
| 通知列表 | P1 | 查看历史通知记录 |
| 标记已读 | P2 | 标记单条或全部通知为已读 |
| 未读数量 | P2 | 显示未读通知数量 |

#### 3.2.6 管理员模块

| 功能点 | 优先级 | 描述 |
|--------|--------|------|
| 任务管理 | P1 | 查看所有任务，下架违规任务 |
| 争议处理 | P1 | 查看争议详情，判定结果 |
| 用户管理 | P2 | 冻结/解冻用户账户 |
| 数据统计 | P2 | 平台运营数据统计 |

## 4. 非功能需求分析

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 系统响应时间 | API响应<200ms，页面加载<2s |
| 并发用户数 | 支持100+同时在线 |
| 可用性 | 99.9%正常运行时间 |

### 4.2 安全需求

| 需求 | 描述 |
|------|------|
| 身份认证 | JWT无状态令牌认证 |
| 密码加密 | BCrypt加密存储 |
| 权限控制 | 基于角色的访问控制(RBAC) |
| 数据校验 | 后端参数全面校验 |
| SQL注入防护 | MyBatis-Plus参数化查询 |

### 4.3 可靠性需求

| 需求 | 描述 |
|------|------|
| 事务一致性 | 资金操作必须原子性 |
| 数据完整性 | 外键约束、唯一约束 |
| 日志追踪 | 关键操作记录审计日志 |

## 6. 数据分析

### 6.1 主要数据实体

| 实体 | 说明 | 主要字段 |
|------|------|----------|
| User | 用户信息 | 手机号、密码、角色、信用分 |
| Task | 任务信息 | 标题、奖励、状态、发布者、承接者 |
| Account | 账户信息 | 用户ID、余额、冻结金额 |
| Transaction | 交易记录 | 类型、金额、关联任务 |
| Review | 评价记录 | 评分、内容、评价人、被评价人 |
| Notification | 通知记录 | 类型、内容、已读状态 |
| Dispute | 争议记录 | 原因、结果、关联任务 |
| TaskLog | 任务日志 | 操作类型、描述、时间 |

|----------|--------|------------|
| 用户 | 50 | 18000 |
| 任务 | 200 | 73000 |
| 交易 | 400 | 146000 |
| 评价 | 200 | 73000 |

## 7. 验收标准

### 7.1 功能验收

- [ ] 用户可以完成注册、登录操作
- [ ] 需求方可以发布任务并冻结资金
- [ ] 接单方可以承接任务并提交完成凭证
- [ ] 需求方可以验收任务并触发资金结算
- [ ] 双方可以互发评价并更新信用分
- [ ] 用户可以发起争议并等待处理
- [ ] 管理员可以处理争议、统计数据

### 7.2 非功能验收

- [ ] 系统API响应时间<200ms
- [ ] 资金操作符合ACID特性
- [ ] 用户密码加密存储
- [ ] JWT令牌认证有效
- [ ] 实时通知推送正常
