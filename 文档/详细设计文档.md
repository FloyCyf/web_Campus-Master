#### 1.1.2 核心方法逻辑

**注册方法流程**：
```
1. 验证手机号是否已注册
2. 创建用户对象，密码加密存储
3. 设置默认信用分100，状态正常
4. 插入用户记录
5. 创建用户账户（初始化余额为0）
6. 返回用户信息
```

**登录方法流程**：
```
1. 根据手机号查询用户
2. 验证密码是否正确
3. 生成JWT Token（包含userId, role, expireTime）
4. 返回Token和用户信息
```

#### 1.2.2 任务状态机

```
                    ┌─────────────┐
                    │   pending   │  待接单
                    └──────┬──────┘
                           │ 接单
                           ▼
                    ┌─────────────┐
         ┌─────────│   ongoing   │  进行中
         │         └──────┬──────┘
         │                │ 提交凭证
         │                ▼
         │         ┌─────────────┐
         │         │pending_review│ 待验收
         │         └──────┬──────┘
         │                │ 验收确认
         │                ▼
         │         ┌─────────────┐
         │         │  completed  │  已完成
         │         └─────────────┘
         │
    取消任务          发起争议
         │                ▼
         │         ┌─────────────┐
         │         │  disputed   │  争议中
         │         └──────┬──────┘
         │                │ 管理员裁决
         ▼                ▼
   ┌───────────┐   ┌───────────┐
   │ cancelled │   │ completed │
   │  (取消)   │   │  (完成)   │
   └───────────┘   └───────────┘
```

#### 1.2.3 核心方法逻辑

**创建任务**：
```
1. 验证奖励金额>0
2. 计算服务费（奖励*5%）
3. 冻结需求方账户资金（奖励+服务费）
4. 创建任务记录，状态=pending
5. 记录任务日志
6. 发送通知给相关用户
7. 返回任务ID
```

**接单任务**：
```
1. 检查任务状态是否为pending
2. 检查接单方不是任务发布者
3. 检查接单方信用分>=60
4. 使用乐观锁更新任务状态为ongoing
5. 设置helperId和acceptTime
6. 记录任务日志
7. 发送通知
```

**确认完成**：
```
1. 验证当前用户是任务发布者
2. 验证任务状态是pending_review
3. 解冻需求方资金，转入接单方账户
4. 更新任务状态为completed
5. 更新任务完成时间
6. 记录资金交易日志
7. 记录任务日志
8. 发送通知
```

### 1.4 评价模块

#### 1.4.1 评价计算规则

```java
// 信用分计算
public int calculateCreditScore(int currentScore, int rating) {
    // 好评(5星)：+2分
    // 良好(4星)：+1分
    // 中评(3星)：+0分
    // 差评(1-2星)：-5分
    // 上限：100分
    // 下限：0分

    int delta = switch (rating) {
        case 5 -> 2;
        case 4 -> 1;
        case 3 -> 0;
        default -> -5;
    };

    return Math.max(0, Math.min(100, currentScore + delta));
}
```

## 4. 异常处理设计

### 4.1 异常类定义

```java
public class BusinessException extends RuntimeException {
    private Integer code = 400;
    private String message;

    public static BusinessException badRequest(String message) {
        return new BusinessException(400, message);
    }

    public static BusinessException unauthorized(String message) {
        return new BusinessException(401, message);
    }

    public static BusinessException forbidden(String message) {
        return new BusinessException(403, message);
    }

    public static BusinessException notFound(String message) {
        return new BusinessException(404, message);
    }
}
```

### 4.2 全局异常处理

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    public Result<Void> handleBusinessException(BusinessException e) {
        return Result.error(e.getCode(), e.getMessage());
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public Result<Void> handleValidException(MethodArgumentNotValidException e) {
        String message = e.getBindingResult().getFieldError().getDefaultMessage();
        return Result.error(400, message);
    }

    @ExceptionHandler(Exception.class)
    public Result<Void> handleException(Exception e) {
        log.error("系统异常", e);
        return Result.error(500, "系统异常，请稍后重试");
    }
}
```

| 403 | 权限不足 |

## 5. 事务管理设计

### 5.1 事务边界

| 操作 | 事务范围 |
|------|----------|
| 创建任务 | 冻结资金 + 创建任务记录 + 记录日志 |
| 接单任务 | 更新任务状态 + 记录日志 |
| 确认完成 | 解冻资金 + 增加余额 + 更新任务状态 + 记录日志 |
| 充值 | 增加余额 + 记录日志 |
| 争议处理 | 更新任务状态 + 解冻/扣除资金 + 记录日志 |

### 5.2 事务传播行为

所有Service方法使用 `@Transactional(propagation = Propagation.REQUIRED)`

### 5.3 事务回滚条件

- 抛出 `BusinessException` 及其子类
- 抛出任何 `RuntimeException`
- 抛出检验不通过的 `Exception`
