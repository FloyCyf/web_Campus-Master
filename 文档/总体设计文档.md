# 总体设计文档

## 1. 项目概述

### 1.1 项目名称

校园"万事达"互助众包任务平台（Campus Master）

### 1.2 项目类型

前后端分离的Web应用系统

### 1.3 技术栈概览

| 层级 | 技术选型 | 版本 | 说明 |
|------|----------|------|------|
| 前端框架 | Vue 3 | 3.4.21 | 渐进式JavaScript框架 |
| 前端构建 | Vite | 5.2.0 | 下一代前端构建工具 |
| UI框架 | Tailwind CSS | 3.4.1 | 原子化CSS框架 |
| 状态管理 | Pinia | 2.1.7 | Vue状态管理库 |
| 前端路由 | Vue Router | 4.3.0 | Vue官方路由 |
| 后端框架 | Spring Boot | 3.2.0 | Java企业级应用框架 |
| 数据库 | MySQL | 8.0 | 关系型数据库 |
| ORM框架 | MyBatis-Plus | 3.5.5 | MyBatis增强框架 |
| 安全框架 | Spring Security | 6.1 | Spring安全框架 |
| 认证方案 | JWT | 0.12.5 | 无状态令牌认证 |
| 实时通信 | WebSocket | - | 浏览器实时通信 |
| Java版本 | JDK | 17 | LTS长期支持版本 |

## 3. 功能模块设计

### 3.1 模块划分

| 模块 | 描述 | 依赖关系 |
|------|------|----------|
| 用户模块 | 注册、登录、认证、个人信息 | 基础模块 |

## 4. 数据库设计

| 项目 | 选择 | 原因 |

| 数据库类型 | MySQL 8.0 | 成熟稳定，社区活跃，教育场景首选 |

## 6. 安全设计

### 6.1 认证方案

```
┌─────────────────────────────────────────────────────────────┐
│                      JWT认证流程                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────┐    ┌─────┐    ┌─────┐    ┌─────┐    ┌─────┐     │
│  │用户 │───>│登录 │───>│验证 │───>│生成 │───>│返回 │     │
│  │输入 │    │请求 │    │密码 │    │Token│    │Token│     │
│  └──┬─┘    └──┬─┘    └──┬─┘    └──┬─┘    └──┬─┘     │
│     │         │         │         │         │          │
│     │    ┌────┴────┐    │         │         │          │
│     │    │  验证失败 │    │         │         │          │
│     │    │  401错误 │    │         │         │          │
│     │    └─────────┘    │         │         │          │
│                                                             │
│  ┌─────┐    ┌─────┐    ┌─────┐    ┌─────┐                 │
│  │请求 │───>│携带 │───>│验证 │───>│通过 │                 │
│  │API │    │Token│    │签名 │    │处理 │                 │
│  └──┬─┘    └──┬─┘    └──┬─┘    └──┬─┘                 │
│     │         │         │         │                     │
│     │    ┌────┴────┐    │    ┌────┴────┐                 │
│     │    │ Token   │    │    │ 返回    │                 │
│     │    │ 过期/无效│    │    │ 业务数据│                 │
│     │    │ 401错误 │    │    └─────────┘                 │
│     │    └─────────┘    │                                │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 权限控制

| 角色 | 权限范围 |
|------|----------|
| requester | 发布任务、充值、验收、发起争议、评价接单方 |
| helper | 接单、提交凭证、收取报酬、发起争议、评价需求方 |
| admin | 所有业务数据的查看和管理，争议的最终裁决权 |

### 6.3 数据安全措施

| 措施 | 实现 |
|------|------|
| 密码加密 | BCryptPasswordEncoder |
| SQL注入防护 | MyBatis-Plus参数化查询 |
| XSS防护 | Spring Security响应头配置 |
| CSRF防护 | CORS白名单机制 |

## 8. 技术选型理由

### 8.1 前端技术选型

| 技术 | 选型理由 |
|------|----------|
| Vue 3 | 组合式API更适合复杂业务逻辑，响应式系统优秀 |
| Vite | 极速启动，热更新快，现代化构建工具 |
| Pinia | 官方推荐状态管理，比Vuex更轻量简洁 |
| Tailwind CSS | 原子化CSS，开发效率高，一致性好 |

### 8.2 后端技术选型

| 技术 | 选型理由 |
|------|----------|
| Spring Boot 3 | 生态完善，约定优于配置，内嵌服务器 |
| MyBatis-Plus | 强大的CRUD能力，内置分页，代码生成 |
| JWT | 无状态认证，适合分布式架构 |
| WebSocket | 实时通知推送，替代轮询方案 |

## 9. 性能考虑

### 9.1 前端优化

- Vite按需编译，加快启动速度
- 路由懒加载，减少首屏加载时间
- Pinia状态持久化，避免重复请求

### 9.2 后端优化

- HikariCP连接池，高并发下表现优异
- MyBatis-Plus分页插件，避免全表查询
- 数据库索引优化，常用查询字段建立索引
